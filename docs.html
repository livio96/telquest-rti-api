TelQuest Inventory API

Programmatic access to TelQuest's real-time and inbound inventory. Query Cisco, Avaya, Polycom, and other manufacturer parts straight from NetSuite, with the same data shown on telquestintl.com.

REST · JSON v1.0 5-minute cache
Your API Key
Paste your key here — every curl example below updates automatically.

Introduction

The TelQuest Inventory API is a JSON REST API. Every endpoint accepts and returns standard application/json. All endpoints (except /health) require an API key passed as the X-API-Key HTTP header.

  • Real-time inventory pulled from NetSuite (cached for 5 minutes per request)
  • Inbound inventory sourced from the daily TelQuest CSV feed
  • Filter by part number, manufacturer, condition, price, and stock status
  • Pagination via limit and offset
  • Interactive Swagger spec at /__docs__/

Base URL

All API requests are made to:

https://telquest-rti-api.onrender.com

Authentication

Every request (except /health) must include your API key as the X-API-Key header. Requests missing or with an invalid key receive a 401 Unauthorized response.

Treat your key like a password. Don't commit it to git, don't share it in chat. If a key is compromised, contact TelQuest to rotate it.

Example request

curl -H "X-API-Key: YOUR_API_KEY" \
  "https://telquest-rti-api.onrender.com/inventory?limit=5"
import requests

headers = {"X-API-Key": "YOUR_API_KEY"}
r = requests.get(
    "https://telquest-rti-api.onrender.com/inventory",
    headers=headers,
    params={"limit": 5},
)
print(r.json())
const res = await fetch(
  "https://telquest-rti-api.onrender.com/inventory?limit=5",
  { headers: { "X-API-Key": "YOUR_API_KEY" } }
);
const data = await res.json();
console.log(data);
library(httr)

res <- GET(
  "https://telquest-rti-api.onrender.com/inventory",
  query = list(limit = 5),
  add_headers("X-API-Key" = "YOUR_API_KEY")
)
content(res, "parsed")

Errors

The API uses standard HTTP status codes:

CodeMeaning
200Success.
401Missing or invalid X-API-Key header.
404Endpoint or part number not found.
500Server error — most often the upstream NetSuite call failed. Retry after a few seconds.

Error response shape

{
  "error": "Missing or invalid X-API-Key header"
}

Health check

Returns server status. Used by uptime monitors and load balancers. No authentication required.

GET /health 🔓 No auth

Description

Lightweight liveness probe. Returns immediately without contacting NetSuite or any other dependency. Use this to verify the API is reachable.

Response fields

statusstringAlways "ok" when reachable.
timestringServer timestamp, ISO 8601 with timezone offset.
 
curl "https://telquest-rti-api.onrender.com/health"
200 OK 
{
  "status": "ok",
  "time": "2026-05-15T15:39:41+0000"
}

List inventory

The main endpoint. Returns inventory items matching optional filters, with pagination. Combines real-time NetSuite data with the inbound CSV feed.

GET /inventory 🔒 Requires X-API-Key

Query parameters

All parameters are optional. Filters can be combined.

ParameterTypeDescription
part_numberstringSubstring match, case-insensitive. e.g. WS-C2960.
manufacturerstringExact match, case-insensitive. e.g. Cisco, Polycom.
conditionstringExact match, case-insensitive. One of: New, Like New, Refresh, Recertified, New Open Box.
in_stock_onlybooleantrue excludes inbound items (real-time only).
on_sale_onlybooleantrue returns only items with an active promotion.
min_pricenumberMinimum price (USD). Items with non-numeric prices are excluded.
max_pricenumberMaximum price (USD).
limitintegerMax rows to return. Default 1000.
offsetintegerRows to skip for pagination. Default 0.

Response fields

countintegerNumber of items in this response (≤ limit).
totalintegerTotal matching items before pagination.
fetched_atstringWhen the cached data was last refreshed.
itemsarrayArray of inventory item objects.
 
curl -H "X-API-Key: YOUR_API_KEY" \
  "https://telquest-rti-api.onrender.com/inventory?manufacturer=Cisco&in_stock_only=true&limit=10"
import requests

resp = requests.get(
    "https://telquest-rti-api.onrender.com/inventory",
    headers={"X-API-Key": "YOUR_API_KEY"},
    params={
        "manufacturer": "Cisco",
        "in_stock_only": "true",
        "limit": 10,
    },
)
data = resp.json()
for item in data["items"]:
    print(item["part_number"], item["price"])
const params = new URLSearchParams({
  manufacturer: "Cisco",
  in_stock_only: "true",
  limit: 10,
});

const res = await fetch(
  `https://telquest-rti-api.onrender.com/inventory?${params}`,
  { headers: { "X-API-Key": "YOUR_API_KEY" } }
);
const { items } = await res.json();

Pagination

Use limit and offset together to page through results. The total field in every response tells you how many rows match before pagination, so you can compute total pages.

# Page 1: items 0–99
curl -H "X-API-Key: YOUR_API_KEY" \
  "https://telquest-rti-api.onrender.com/inventory?limit=100&offset=0"

# Page 2: items 100–199
curl -H "X-API-Key: YOUR_API_KEY" \
  "https://telquest-rti-api.onrender.com/inventory?limit=100&offset=100"

Single part lookup

Returns a single inventory item matching an exact part number (case-insensitive). Use this when you already know the SKU.

GET /inventory/{part_number} 🔒 Requires X-API-Key

Path parameter

part_numberstringExact part number, case-insensitive. URL-encode if it contains special characters.

Response

Returns { "found": true, "items": [...] } if matched, or { "found": false, "part_number": "..." } if no match.

curl -H "X-API-Key: YOUR_API_KEY" \
  "https://telquest-rti-api.onrender.com/inventory/WS-C2960-24TC-L"
200 OK 
{
  "found": true,
  "items": [
    {
      "part_number": "WS-C2960-24TC-L",
      "description": "Catalyst 2960 24-port switch",
      "manufacturer": "Cisco",
      "condition": "Refresh",
      "quantity": 12,
      "price": "$185",
      "price_numeric": 185,
      "on_sale": false
    }
  ]
}
200 OK 
{
  "found": false,
  "part_number": "NOT-A-REAL-SKU"
}

Refresh cache

Forces an immediate refresh of the inventory cache from NetSuite. Use sparingly — the cache automatically refreshes every 5 minutes anyway. Intended for situations when you've just updated something in NetSuite and need it visible immediately.

POST /refresh 🔒 Requires X-API-Key

Description

Triggers a fresh pull from NetSuite + the inbound CSV feed, replacing the cached data. The response contains the new refreshed_at timestamp and the row count.

Cost. Each call hits NetSuite directly, which is slower (1–3 seconds) and consumes a SuiteQL query against your governance limits. Don't call this on every page load. 
curl -X POST -H "X-API-Key: YOUR_API_KEY" \
  "https://telquest-rti-api.onrender.com/refresh"
200 OK 
{
  "refreshed_at": "2026-05-15T15:42:08+0000",
  "rows": 3214
}

The inventory item object

Every item returned by the API has the following fields:

FieldTypeDescription
part_numberstringManufacturer SKU. Primary identifier.
descriptionstringHuman-readable product description.
manufacturerstringBrand name (Cisco, Polycom, Avaya, etc.).
conditionstringCondition grade. One of: New, Like New, Refresh, Recertified, New Open Box.
quantityintegerAvailable units. Capped at 250; values above are reported as 250.
pricestringDisplay price as a string with currency prefix, e.g. "$185". May be "Call" for inbound items without pricing.
price_numericnumberNumeric price for filtering and sorting. null if non-numeric.
linkstringPublic product page on telquestintl.com.
promotion_exp_datestringISO date when the current promotion expires, or null.
inventory_typestring"real-time" for ready-to-ship NetSuite inventory; "inbound" for the daily CSV feed.
is_inboundbooleantrue if the item is an inbound (not yet received) entry.
on_salebooleantrue if a promotion is currently active.

Caching

To keep response times fast and limit pressure on NetSuite, the API caches the full inventory dataset for 5 minutes. The first request after expiry triggers a refresh; subsequent requests within the window return cached data.

  • Every response includes a fetched_at field showing cache age.
  • Most queries return in under 200 ms (cache hit).
  • Cache misses take 1–3 seconds (NetSuite query latency).
  • To force an immediate refresh, call POST /refresh.

CORS

The API sets Access-Control-Allow-Origin: *, so you can call it directly from a browser-based JavaScript app. The X-API-Key header is allowed.

Don't ship your API key in browser code. A key embedded in client-side JavaScript is visible to anyone viewing source. For browser apps, proxy through your own backend that holds the key as a server-side secret.
